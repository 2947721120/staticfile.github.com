1. Background & Purpose

As Google Ajax Library, Microsoft ASP.net CDN, SAE, Baidu, Upyun and etc are providing free CDN for JS libs，but most of them only provide a service for JS libs. We find that CDNJS is storing much more libs, JS / CSS / Images / Swf, whereas it is slow in China and DONOT provide libs that created by Chinese developers.

Thus we decide to provide a warehouse to store open-source libs as much as possible, and serve them in a more stable & faster CDN. In addition, we're also providing you an enterance to submit lib(s), including JS / CSS / Images & Swf Files.

2. How To Sumbit A Lib

Fork the repo，create a folder under libs/, the folder should contain at least one lib. Like：

.
├── jquery                    # ... name of the lib
│   ├── 1.8.3                 # ... version
│       └── jquery.min.js     # ... file name
│   ├── 1.9.1
│       └── jquery.min.js
└── package.json              # ... for the latest version

Edit package.json ，a json file contains the infomation of the LATEST version of the lib. Here's a demo file in real-world side：libs/typo.css/package.json.

{
  // please keep the folder name the same as the file name
  "name": "name of lib",
  "filename": "main file of the lib, like XYZ.js",
  "version": "1.0",

  // others
  "description": "description of the lib",
  "homepage": "the homepage of the lib, like http://staticfile.org",
  "keywords": [],
  "maintainers": [
    {
      "name": "author or maintainer",
      "web": "http://staticfile.org",
      "mail": "trimidea@gmail.com"
    }
  ],
  "repositories": [
    {
      // git, svn, hg & etc.
      "type": "git",
      "url": "https://github.com/staticfile/static"
    }
  ]
}

After this, send a Pull Request to us. We will review it as soon as possible. Once the Pull Request is Merged，the files of the lib are accessible.

In order to keep fast and stable, we suggest you only submit libs that already have influence -- like, has 20 stars on Github.

3. Updates

The repo fetches updates from CDNJS everyday and user's pull request. If you can't find the right version at staticfile.org. Please create an issue or send us a Pull Request.
